The Course: Regression Analysis

Before I talk about the infrastructure, here is a brief description of the course to give you some context:

Regression Analysis is a second-semester undergraduate course focused on analyzing multivariate data using regression models. Because the course is early in the undergraduate curriculum, the primary focus is on application rather than the mathematical theory. A more detailed description of the course is at the top of the course website.

There were 90 students in the course in Spring 2020, covering all years - freshmen to senior. The course is designed to include a lecture 2X a week and a lab with 30 students in each lab section.

The course can be described as “technology supported”[LINK?], i.e. most of the course is facilitated online and is anchored by in-person lectures and labs. The infrastructure is based on the one described in Data Science in a Box. I’ll describe each part of the infrastructure below.

Materials

  • .emph[Course website]: The course website is the central hub for the course. Students can access anything they need including the syllabus, course schedule, lecture slides, and assignment instructions from the website. It also includes links to the other platforms we use (e.g. Sakai, GitHub). I encourage students to bookmark the course website, since it is the one-stop shop for everything they need in the course.

  • .emph[Learning Management System (LMS)]: All protected content, such as grades, are put in the LMS. It also includes links to the course schedule, syllabus, and the platforms listed below, in case students would rather navigate the course through the LMS rather than the website.

Assignments

  • .emph[GitHub]: All homework, lab assignments, and the final group project are done using GitHub and RStudio. An “organization” is created for the course on GitHub and is connected to GitHub Classroom so students’ work can remain private. For each assignment, a private repository, i.e. directory, is created for each student or group. The repo contains an R Markdown template, the data, and a link to the assignment instructions. Students connect the repo to a project in RStudio and “push” their updates to the repo as they complete the assignment. I won’t go into more detail here, but Reproducible Workflows: R Markdown, Git, GitHub from the Desgining the Data Science Classroom has more information about the assignment workflow using RStudio and GitHub.

  • .emph[RStudio]: Students use RStudio through a server maintained by the university’s Office of Information Technology. They access RStudio through a web browser, so they can use it on almost any device, including tablets and notebooks. RStudio Cloud is a cloud-based platform provided and maintained by RStudio that is another great served-based option if your institution doesn’t have RStudio servers. The article Infrastructure and tools for teaching computing throughout the statistical curriculum discusses the details and benefits of using RStudio through servers for teaching.

  • .emph[Gradescope]: Students submit their assignments on Gradescope, an online grading and rubric platform. Students can see their assignment grades, grading rubrics, and comments on Gradescope. It can sync with numerous LMS platforms, so the grades can easily be posted in the gradebook in the LMS.

To get an idea of how I use RStudio, GitHub, and Gradescope in a typical assignment, check out the slides from my Women in Statistics and Data Science talk Using GitHub and RStudio to facilitate authentic learning experiences in a regression analysis course.

Communication

  • .emph[Student Q&A]: Students use Piazza, an online Q&A platform to post questions about assignments and the course content. Anyone in the course can respond to the question, so this gives students an opportunity to answer each others’ questions. It also keeps me and the teaching assistants from having to answer the same question multiple times.

  • .emph[Teaching Team Communication]: I use Slack to communicate with the teaching team. We have multiple channels - general, hw, labs, exams, project, office-hours - to help organize the conversation. Aside from an initial email at the beginning of the semester, all communication with the teaching team occurs in Slack. This helps keep everything in one place, and the more relaxed nature of Slack (compared to email) encourages more frequent discussion.
