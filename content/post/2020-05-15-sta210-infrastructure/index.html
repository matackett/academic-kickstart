To give you can idea of what a technology-enhanced course looks like, here is a description of how students interact what they need for the course:

  • Course website: Everything needed for the course is accessible from the website. This includes all course content along with links to Sakai, GitHub, and RStudio.

  • Sakai: This is Duke‚Äôs Learning Managment System (LMS). It is the place for all protected content such as grades. A link to Sakai is available on the course website.

  • GitHub: All homework, lab assignments, and the final group project are done using GitHub and RStudio. A private repo is created for each student (or group) that contains an R Markdown template, the data, and a link to the assignment instructions. Students push their updates to the repo as they complete the assignment.   • Gradescope: Students submit their assignments on Gradescope, an online grading and rubric platform. Students can see their assignment grade, the grading rubric, and comments on Gradescope; it is synced with Sakai, so the assignment grade also gets posted in the gradedbook in Sakai.

  • Piazza: Online forum for asking questions.

Since so much of the course was already operating online, my primary focus during the transition was figuring out how to move the face-to-face interactions from the classroom to a video conference. In other words, how to do lectures, labs, office hours, and the final group project on Zoom. This meant not only figuring out how to use all the features in Zoom but also how to engage students synchronously and asychronously.
